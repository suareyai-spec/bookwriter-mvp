Authentication

All endpoints (except /api/health) require an API key in the Authorization header:

Authorization: Bearer YOUR_API_KEY

Test your connection:

curl https://api.narevo.ai/api/health
→ {"status":"ok","timestamp":"2026-02-12T00:00:00.000Z"}

Accounts

POST /api/accounts/connect

Connect a social media account by storing OAuth tokens.

{
  "platform": "instagram",
  "platformAccountId": "17841400123456",
  "accountName": "mybrandofficial",
  "accessToken": "IGQVJWZAxxxxxxxxx...",
  "refreshToken": "IGROxxxxxxxxx...",
  "tokenExpiresAt": "2026-04-01T00:00:00Z"
}
Tokens are AES-256 encrypted at rest and never returned in API responses.
FieldTypeRequiredDescription
platformstringYesfacebook, instagram, youtube, tiktok
platformAccountIdstringYesAccount ID from the platform
accountNamestringYesDisplay name
accessTokenstringYesOAuth access token
refreshTokenstringNoOAuth refresh token
tokenExpiresAtISO dateNoToken expiration
GET /api/accounts

List all connected accounts. Tokens are excluded from response.

DELETE /api/accounts/:id

Disconnect a social account and delete stored tokens.

POST /api/accounts/:id/refresh

Refresh an expired OAuth token using the stored refresh token.

Posts

POST /api/posts/schedule

Schedule a post for future publishing.

{
  "accountId": "abc123-uuid",
  "platform": "instagram",
  "content": "Excited to announce our summer collection!",
  "mediaUrls": ["https://cdn.example.com/photo1.jpg"],
  "thumbnailUrl": null,
  "scheduledAt": "2026-02-15T14:00:00Z"
}

The API automatically publishes at the scheduled time via BullMQ job queue with 3 retries on failure.

FieldTypeRequiredDescription
accountIdstringYesID from /api/accounts
platformstringYesfacebook, instagram, youtube, tiktok
contentstringYesPost caption/text
mediaUrlsstring[]NoArray of media URLs
thumbnailUrlstringNoCustom thumbnail
scheduledAtISO dateYesWhen to publish (UTC)

Post statuses: draft → scheduled → publishing → published or failed

POST /api/posts/publish

Publish a post immediately. Same body as schedule (without scheduledAt).

GET /api/posts

List posts with optional filters.

Query ParamDescription
statusFilter: draft, scheduled, publishing, published, failed
platformFilter: instagram, facebook, youtube, tiktok
startDateFilter: YYYY-MM-DD
endDateFilter: YYYY-MM-DD
GET /api/posts/:id

Get a single post by ID.

PUT /api/posts/:id

Update a scheduled or draft post. Cannot update published posts.

DELETE /api/posts/:id

Cancel and delete a post.

AI Captions

POST /api/captions/generate

Generate platform-optimized captions with hashtags using AI.

{
  "topic": "summer fashion collection launch",
  "platform": "instagram",
  "tone": "engaging",
  "length": "medium",
  "count": 3
}

Response

{
  "captions": [
    {
      "text": "Summer just got better. Our new collection drops NOW...",
      "hashtags": ["#SummerFashion", "#NewCollection", "#OOTD"]
    }
  ]
}
FieldOptionsDefault
topicAny descriptive stringrequired
platforminstagram, facebook, youtube, tiktokrequired
toneprofessional, casual, funny, engagingengaging
lengthshort, medium, longmedium
count1–53
Captions are optimized per platform: Instagram = hashtag-heavy, TikTok = trendy/short, YouTube = SEO-friendly, Facebook = conversational.

Rate Limits

EndpointLimit
POST /api/posts/*100 requests/hour
POST /api/captions/generate50 requests/hour
GET /api/analytics/*200 requests/hour
All other endpoints1000 requests/hour

Every response includes rate limit headers:

X-RateLimit-Limit: 100
X-RateLimit-Remaining: 87
X-RateLimit-Reset: 1707700800
When exceeded, the API returns 429 Too Many Requests with a Retry-After header (seconds).

Error Handling

{ "error": "Description of what went wrong" }
StatusMeaning
400Bad request — missing or invalid fields
401Unauthorized — missing or invalid API key
403Forbidden — IP not in allowlist
404Resource not found
429Rate limit exceeded
500Internal server error

Security

  • ✓ AES-256-GCM encryption — All OAuth tokens encrypted at rest
  • ✓ HTTPS only — TLS 1.2+ via Let's Encrypt
  • ✓ Audit logging — All mutating requests logged (who, what, when, IP)
  • ✓ IP allowlisting — Optional per-API-key IP restrictions
  • ✓ Rate limiting — Per-key, per-endpoint limits
  • ✓ Security headers — HSTS, nosniff, frame deny
  • ✓ BullMQ job queue — Scheduled posts survive server restarts

OAuth Setup Guide

Your frontend handles the OAuth redirect flow for each platform. Once the user authorizes, send the tokens to POST /api/accounts/connect.

Meta (Facebook + Instagram)

1. Redirect user to:
   https://www.facebook.com/v18.0/dialog/oauth
   ?client_id={META_APP_ID}
   &redirect_uri={YOUR_CALLBACK}
   &scope=pages_manage_posts,instagram_basic,instagram_content_publish

2. Exchange code for token:
   POST https://graph.facebook.com/v18.0/oauth/access_token

3. Exchange short-lived token for long-lived (60 days)

4. Send tokens to POST /api/accounts/connect

YouTube

1. Redirect user to:
   https://accounts.google.com/o/oauth2/v2/auth
   ?client_id={CLIENT_ID}
   &redirect_uri={YOUR_CALLBACK}
   &scope=youtube.upload,yt-analytics.readonly

2. Exchange code for tokens:
   POST https://oauth2.googleapis.com/token

3. Send tokens to POST /api/accounts/connect

TikTok

1. Redirect user to:
   https://www.tiktok.com/v2/auth/authorize/
   ?client_key={CLIENT_KEY}
   &redirect_uri={YOUR_CALLBACK}
   &scope=video.upload,video.list

2. Exchange code for tokens:
   POST https://open.tiktokapis.com/v2/oauth/token/

3. Send tokens to POST /api/accounts/connect
